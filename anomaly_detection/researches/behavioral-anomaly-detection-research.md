Behavioural Anomaly Detection in the MITRE ATT&CK Framework: A Rigorous Telemetry and Modelling AnalysisExecutive SummaryThe contemporary enterprise threat landscape has evolved beyond the detection capabilities of static signatures and fixed-threshold alerting. Modern adversaries increasingly utilise living-off-the-land (LotL) techniques, valid account abuse, and subtle administrative manipulations that are syntactically indistinguishable from legitimate operations. This report delivers an exhaustive, evidence-backed framework for mapping behavioural anomaly detection to the MITRE ATT&CK Enterprise matrix. By leveraging contextual baselines—including self-baselines, peer-group analytics, relationship graphs, and temporal sequencing—security operations can transition from binary signature matching to continuous, probabilistic risk scoring.The release of MITRE ATT&CK v19 (April 2026) marks a fundamental paradigm shift in how defensive operations must model adversary evasion. The retirement of the legacy Defense Evasion tactic and its bifurcation into Stealth (TA0005) and Defense Impairment (TA0112) establishes a clear operational mandate for detection engineering. Anomaly detection is exceptionally well-suited for Stealth—identifying behavioural camouflage, masquerading, and obfuscation—because the underlying telemetry pipelines remain intact, providing the data necessary to identify deviations from established norms. Conversely, Defense Impairment (e.g., actively disabling Endpoint Detection and Response (EDR) agents or clearing event logs) manifests as a sudden cessation of telemetry. While a drop in volume to zero is technically an anomaly, such techniques are more reliably detected using deterministic integrity validations and heartbeat monitoring rather than complex machine learning models.The integration of User and Entity Behavior Analytics (UEBA) directly into Security Information and Event Management (SIEM) systems demonstrates the increasing operational necessity of anomaly detection. By deriving dynamic features—such as calculating the Term Frequency-Inverse Document Frequency (TF-IDF) of a user's peer group to determine access rarity, or modelling the causal sequences of API calls in cloud environments—defenders can surface high-fidelity alerts that single-event analysis would miss. Advanced detection now relies on graph theoretical approaches to identify exceptional path lengths in privilege escalation and temporal sequence modelling to track protocol state transitions.However, anomaly detection is not universally applicable. Environments lacking stable baselines, such as rapidly scaling ephemeral cloud workloads, suffer from the "cold start" problem, where models lack sufficient historical data to define normalcy, resulting in high false-positive rates. Furthermore, detecting the initial exploitation of known vulnerabilities (e.g., T1190) relies on predictable, deterministic indicators that are more efficiently addressed by signature-based Network Intrusion Detection Systems (NIDS). Ultimately, the success of behavioural anomaly detection requires full-fidelity, schema-on-read telemetry ingestion and rigorous entity resolution across endpoint, identity, network, and cloud perimeters.Anomaly TaxonomyTo accurately map adversary behaviours to detection opportunities, it is necessary to categorise detections by the specific dimensional deviation they measure. Because anomaly models require historical context to trigger, the validation framework must incorporate background noise generation.
