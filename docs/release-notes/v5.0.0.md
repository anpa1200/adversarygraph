# AdversaryGraph v5.0.0 Release Notes

AdversaryGraph v5.0.0 is the Attack Simulation release.

## Added

- Attack Simulation navigation and Discover launchers.
- TTP-first simulation selection flow.
- Per-TTP attack simulation configuration route:
  `/attack-simulation/:simulationId`.
- Local telemetry web server on `127.0.0.1:8765`.
- Docker Compose `attack-lab-web` target container for real lab web attack flows.
  The API sends real HTTP requests to `http://attack-lab-web:8080`, and the
  target container writes access, security, error, auth, and structured JSONL
  logs to the shared Attack Simulation log directory.
- Safe request sets for:
  - `T1595` HTTP/TLS service fingerprint validation.
  - `T1190` public web exposure validation.
  - `T1595` web content discovery and path enumeration.
  - `T1190` path traversal, SQLi/XSS, and SSRF-shaped canaries.
  - `T1059` web command-execution-shaped canaries.
  - `T1505.003` web-shell-shaped URI and POST canaries.
  - `T1105` upload/download transfer canaries.
  - `T1552.001` exposed secret/config/key path canaries.
  - `T1110` web failed-login sequence canaries.
  - `T1110.001` lab-only web login brute-force sequence.
  - `T1110.003` lab-only web password-spray sequence.
  - `T1589.002` lab-only web user-enumeration sequence.
  - `T1071.001` HTTP beacon canaries.
  - `T1041` web exfiltration-shaped POST canaries.
- JSONL telemetry logs for attack runs and web access.
- Web access telemetry now records clean path, query keys, body length, body
  SHA-256, short body preview, and matched canary classifications.
- Real webserver-style logs now include access, WAF/security, error, and auth
  streams. Auth logs include username, user-exists flag, outcome, failure reason,
  attack type, password length/hash, run ID, and simulation ID without cleartext
  password logging.
- Real-time attack log panel.
- SIEM forwarding panel for HTTP(S) collectors with IP/URL destination input,
  direct/Docker-host/auto connection routing, none/bearer/token/basic/
  custom-header authentication, and delivery status.
- Recent SIEM destination history for the last 10 non-secret forwarding targets.
- Backend log-tail and telemetry-forwarding APIs.
- AI Attack Assistant for selected-TTP, threat-actor, and Challenge Me
  telemetry generation workflows.
- 25 named coherent scenario templates with preconditions, success criteria,
  expected detections, and ordered kill-chain phases.
- Complicated attack mode for longer multi-source telemetry across web, WAF,
  DNS, proxy, firewall, Windows Security, Sysmon, and EDR-shaped events.
- Attack-chain graph showing phase order, ATT&CK technique, telemetry source,
  event format, event count, and detection purpose.
- Explain Attack panel for generated scenario rationale, assumptions, and
  detection focus.
- Attack Simulation screenshot pack under
  `docs/assets/attack-simulation-v5/`.

## Changed

- Renamed the former External Simulation UI to Attack Simulation.
- `/external-simulation` now redirects to `/attack-simulation`.
- README release metadata now points to v5.0.0.
- Platform documentation now links the Attack Simulation visual walkthrough and
  screenshot manifest.

## Safety

- Attack Simulation remains lab-scoped and allowlisted.
- The runner does not execute exploit payloads or arbitrary commands. Canary
  values are fixed benign strings and the lab web server records them as
  telemetry only.
- SIEM forwarding rejects unsafe schemes, URL-embedded credentials, and
  metadata/link-local destinations.
